Repères historiques

  • 1989: Tim Berners-Lee et Robert Cailliau inventent le Web.
    Internet est une infrastructure de réseaux munie de moyens de transmission (les routeurs) et de moyens logiciels permettant à des machines de communiquer entre elles.
    Au début d'Internet, il était difficile de trouver une information donnée sur les réseaux. Aussi, deux chercheurs du CERN de Genève, Tim Berners-Lee et Robert Cailliau ont-ils imaginé un ensemble de protocoles et de logiciels que Robert Cailliau aurait voulu appeler Loki, du nom d'un dieu nordique, fourbe, immoral et manipulateur. C'est la naissance du Web qui s'appellera finalement World Wide Web, ils élaborent les trois caractéristiques essentielles du Web:
    • Le protocole HTTP
    • Le langage HTML
    • Le système URL
  • 1993: Il existe seulement 623 sites web. Aujourd'hui, il y a plus de 200 000 000 sites web actifs.
    La diffusion gratuite de ces nouveaux outils propulse le Web sur les réseaux Internet. Le logiciel Mosaïc, développé sous UNIX, est le premier navigateur web doté d'une interface graphique. De l'interactivité avec l'utilisateur est ajoutée grâce aux formulaires qui préfigurent l'essor du commerce en ligne.
    Une architecture mondiale s'impose alors rapidement selon le modèle client-serveur, en particulier grâce à l'entreprise américaine CISCO, qui fiabilise les échanges sur le réseau Internet par la fabrication de routeurs plus fiables et plus rapides, et l'amélioration du protocole de communication TCP/IP. La démocratisation des ordinateurs personnels contribue également à un développement rapide.
  • 1994: Cookies, chiffrement SSL et JavaScript
    Le navigateur web Netscape, dont Mozilla Firefox est l'héritier, apporte une avancée déterminante au Web: la gestion des cookies, la sécurisation des transmissions grâce au protocole SSL et le langage Javascript qui apporte de l'interactivité aux pages grâce à l'écriture de scripts côté client.
  • 1995: Le langage de description PHP (côté serveur) est popularisé par l'écriture de pages web interactives.
  • 1196: La société Netscape introduit JavaScript dans son navigateur web, les pages deviennent dynamiques.
  • 1997: Wi-Fi, haut débit et commerce en ligne se développent
    La norme Wi-Fi 802.11 rend possible la création de réseaux locaux sans fil à haut débit. La consultation des pages web est encore facilité par la suite par l'utilisation croissante des smartphones et des tablettes sans fil.
    Le commerce en ligne se développe avec un chiffre d'affaires d'envrions 100 milliards d'euros en 2019 en France. 80% des internautes ont fait un achat sur le Web en 2019, dont près de la moitié sur leur smartphone, pour un taux de satisfaction de 90%.

Le modèle client-serveur

Le modèle client-serveur est une architecture de communication entre deux machines le plus souvent distinctes: le client et le serveur.

  • Client :
    Le client est la machine (ordinateur, smartphone, etc...) qui demande l'exécution d'un service (comme par exemple, l'affichage d'une page web). Le client envoie des requêtes au serveur. Les clients web sont souvent des navigateurs web.
  • Serveur :
    Un serveur est une machine robuste et rapide chargée de répondre à la demande (requête) du client en lui fournissant le service demandé.
  • Requête : La requête est le message transmis par le client au serveur, décrivant le service à rendre.
  • Réponse : La réponse est le message transmis par le serveur au client suita à l'exécution d'un service.
Dans le modèle client-serveur, cleint et serveur ont des rôles bien distincts:
  • Après l'envoie d'une requête HTTP, le client se met en pause, en attendant la réponse du serveur.
  • Le serveur renvoie une réponse au client, composées d'un code d'état (1xx, 2xx, 3xx, 4xx ou 5xx) et du contenu demandé si la requête est un succès.
    • Attention, ne pas confondre le modèle client-serveur avec le réseaux pair-à-pair dans lequel client et serveur jouent le même rôle.

Description d'une page web

Le Web est un ensemble de pages reliées par des liens hypertexte.
L'affichage d'une page web peut être optimisé (rapidité et présentation) selon les particularités du client (ordinateur, smartphone).

Le langage HTML permet de structurer le contenu de la page web. Il autorise une certaine souplesse de syntaxe: il est alors parfois possible d'ouvrir une balise HTML sans la fermer ou de donner la valeur d'un attribut entre guillemets ou apostrophes. Il est donc possible que la page web n'ait pas toujours le même aspect visuel, selon le navigateur web utilisé.

Le langage CSS est utilisé pour donner du style à la page, pour mettre en forme le texte: centrer un texte ou une image, mettre en gras, changer la police de caractère...

Cette feuille de style définit les choix graphiques effectués par l'auteur de la page. Ainsi, en modifiant l'unique feuille de style externe d'un site web, stockée côté serveur, l'ensemble des pages web se trouve instantanément modifié.

Il est possible de rendre une page web interactive, de telle sorte qu'il y ait de l'interactivité entre l'utilisateur et la page web.
Son contenu pourra alors varier selon l'heure de la requête, le nom de l'utilisateur ou encore le remplissage d'un formulaire par l'utilisateur, etc.
Remarque: La technologie AJAX permet le transfert de données en arrière-plan entre le client et le serveur, sans avoir à recharger la page web. Les suggestions de recherche sur Google utilisent cette technologie.

Scripts côté serveur et côté client

Le Web est basé sur un principe simple: tous les contenus sont fournis par les serveurs web et peuvent être récupérés à distance par les clients.
Ainsi un contenu statique sera-t-il simplement téléchargé par le client, tandis qu'un contenu dynamique ne pourra fonctionner que grâce à l'existence de scripts, des petits bouts de code chargés d'exécuter une action prédéfinie programmée à l'avance. Ces scripts peuvent ainsi être exécutés soit du côté serveur soit du côté client.

Les scripts côté serveur nécessitent une communication avec le serveur et ne permettent pas une interactivité immédiate. L'exécution côté serveur est souvent nécessaire, pour gérer les droits des utilisateurs par exemple.
Un script côté serveur sera par exemple utilisé pour collecter des données via un formulaire ou mettre à jour une base de données.
Le serveur sera donc fortement sollicité.*
Le langage de script côté serveur le plus couramment utilisé de nos jours est le langage de programmation PHP.
Le script sera alors directement intégré en langage PHP, et le code source sera entièrement masqué au client. Lors de la génération de la page web dynamique, c'est une page HTML qui sera transmise au client.

Les scripts côté navigateur s'exécutent directement sur la machine. Il est donc possible d'avoir une intéractivité immédiate, pour des jeux par exemple.
Un script côté client sera exécuté par le client, dans le navigateur web, pour apporter un côté dynamique au site web.
Le script sera donc directement intégré dans le code HTML de la page web ou alors inséré dans un fichier externe. Les machines clientes étant généralement relativement puissantes, cette exécution de scripts côté client permet de soulager les serveurs sans ralentir le processus global.
Bloquer l'exécution d'un script sera cependant toujours possible via une extension installée dans le navigateur web, comme NoScript de Firefox.

JavaScript est un langage utilisé pour les scripts côté client. Il est interprété côté client par le navigateur.

Composants graphiques d'une page web

Les composants usuels

Pour rendre une page web interactive, il faut y insérer des composants d'interface graphique avec lesquels l'utilisateur peut interagir, en agissant dessus avec le doigt (écran tactile) ou la souris (ordinateur).
Ces composants sont compatibles avec les navigateurs les plus utilisés.

Il existe un grand nombre de composants graphiques, dont voici les plus courants:
  • Zone de texte
  • Bouton poussoir
  • Bouton radio
  • Zone de mot de passe
  • Case à cocher
En HTML, chaque composant graphique peut être inséré avec la balise suivante:

<input type="text" value="Hello world !">

type et value étant les attributs de la balise
"text" et "Hello world !" étant les valeurs de ces attributs

L'attribut value

L'attribut value définit la valeur par défaut qui sera affichée dans l'élément au chargement de la page web. Cette valeur est accessible et modifiable, notamment grâce à la méthode getElementById() (JavaScript).

L'attribut id (identifiant unique)

Le côté dynamique de la page web est assuré par la possible modification des valeurs des attributs des composants graphiques.
Pour pouvoir accéder indépendamment à chacun des composants, il faut pouvoir les identifier de manière unique: c'est le rôle de l'attribut id. Chaque composant peut posséder un identifiant unique, noté id, composé d'une chaîne de caractères.

Exemple:

<input type="text" value="Hello !" id="texte1">

Chaque composant devient alors manipulable avec un script. On peut aussi modifier l'aspect d'un composant avec du code CSS.

Le Document Object Model (DOM)

Le DOM (Document Object Model) d'une page HTML est la présentation de la page sous forme d'objets qu'il est possible de manipuler. On peut le modéliser avec la structure d'un arbre, composé de branches et de feuilles, qui représentent les "parents" et les "enfants".
  • Les objets du DOM peuvent représenter une fenêtre, un document, une phrase, un style, etc...
  • Le DOM représente le document affiché par une structure en arbre, comportant des noeuds (branches et feuilles)
  • Le DOM est présent dans le navigateur. Sa manipulation peut être facilitée par des bibliothèques JavaScript, comme par exemple, jQuerry.

La méthode getElementById(id)

La méthode getElementById(id) est l'une des fonctions les plus utiles du DOM HTML, car elle permet de renvoyer l'objet dont l'identifiant id est donné en paramètre de la fonction.
Il est alors possible de modifier par un script les attributs de l'objet: la page devient dynamique.

Exemple:

document.getElementById("texte1").value;
renvoie "Hello !": c'est la valeur de l'attribut value du composant ayant pour identifiant "texte1".

document.getElementById("texte1").value = "Bonjour";
change le contenu du composant ayant pour identifiant "texte1" et le remplace par "Bonjour".
